# Skill 准入规则

## 硬门槛

任一项命中时，总分不得将结论提升到“建议准入”：

1. `SKILL.md` 缺失、frontmatter 无效，或文件夹名与 `name` 不一致。
2. 存在凭据、Token、私钥、个人隐私、本地敏感绝对路径或不应发布的原始材料。
3. 引用的必需文件、脚本或资产缺失。
4. 存在高风险、破坏性、对外发布或写操作，但没有目标确认、授权、停止条件或失败处理。
5. 核心工作流无法从输入走到明确交付物。
6. 输出或建议可能误导用户，且没有事实、证据或不确定性标记。
7. 用户要求公开或团队准入，但核心脚本/产物从未运行，且没有可接受的人工验收记录。

## 准入结论

| 结论 | 条件 | 含义 |
|---|---|---|
| A 建议准入 | 无硬门槛，总分 `>= 85`，且 D1、D2、D4、D5、D9 都 `>= 4/5` | 可按目标范围使用或发布 |
| B 有条件准入 | 无硬门槛，总分 `70-84`，且关键维度都 `>= 3/5` | 可限于个人/小范围试用，完成 P0/P1 后再公开 |
| C 修改后复评 | 总分 `50-69`，或关键维度任一 `< 3/5` | 不建议当前版本准入，先修改再重评 |
| D 不建议准入 | 命中硬门槛，或总分 `< 50` | 当前版本不应安装、分发或作为稳定工作流使用 |

## 目标范围修正

- **个人使用**：可接受少量环境假设，但不得跳过安全、隐私和核心流程验证。
- **团队使用**：必须说清依赖、安装、输入、输出、失败处理和升级方式。
- **公开发布**：还必须完成隐私/凭据扫描、正反触发样例、代表性工作流验证和安装路径验证。

## 优化优先级

- `P0`：对应硬门槛，不修复不得准入。
- `P1`：明显影响触发、核心工作流、输出正确性或可验证性。
- `P2`：影响可维护性、资源组织、写作和局部体验。
- `P3`：非必要改进，不影响当前准入。

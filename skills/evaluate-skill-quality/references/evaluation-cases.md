# 评估 Skill 的测试样例

这组样例用于检查 `evaluate-skill-quality` 自身的触发边界和工作流。测试时必须记录“已执行”或“仅设计”。

## 应触发

1. “按 D1-D9 评估这个 Skill，看能不能公开发布。”
2. “这个 Skill 刚生成，请给出评分、准入结论和 Agent 修改任务。”
3. “对比这个 Skill 修改前后的质量，用同一组 case 复评。”

## 不应触发

1. “帮我安装这个 Skill。”应使用 `skill-installer`。
2. “从零创建一个 PDF 编辑 Skill。”应主要使用 `skill-creator`；创建完成后再评估。
3. “检查这个 Python 项目的代码质量。”目标不是 Skill 包，不应触发。

## 正常工作流

1. **新 Skill 准入**：目标包含有效 `SKILL.md`、接口元数据、引用资源和脚本。预期输出结构校验、D1-D9 证据评分、硬门槛、A-D 准入结论与可验收优化任务。
2. **修改后复评**：提供旧版分数、新版文件和相同 case。预期只对有新证据的维度提分，列出前后差异和剩余风险。

## 失败与边界

1. **目标路径不存在**：停止评分，要求用户给出正确路径；不根据 Skill 名称猜测内容。
2. **脚本无法安全运行**：标记“未验证”，按已有证据评分，并将隔离测试列为准入前任务。
3. **发现凭据或隐私**：触发硬门槛，结论为 D；不在报告中回显完整敏感值。

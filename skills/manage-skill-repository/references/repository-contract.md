# 仓库组件矩阵

维护时检查受影响的所有组件，不只检查 `SKILL.md`。

## 根目录

| 组件 | 维护要求 |
|---|---|
| `README.md` | 目录、用途、源码链接、Codex 安装提示词、终端命令、仓库树与实际一致 |
| `.gitignore` | 排除缓存、密钥、虚拟环境、本地输出和系统文件；不忽略 Skill 必需资产 |
| `LICENSE` | 新增/修改 Skill 引入第三方资源时检查许可影响；未授权不自动改变仓库许可证 |
| 仓库级自动化 | 如存在 CI/工作流，检查触发条件、权限、密钥引用、校验范围和失败行为 |
| Git 状态 | 分支、远程、上游、未提交更改、本地/远程分歧和最新提交可解释 |

## Skill 目录

| 组件 | 维护要求 |
|---|---|
| `SKILL.md` | frontmatter 只包含允许字段；`name` 与目录名一致；`description` 说清能力与触发场景；正文有端到端流程、边界、失败处理和资源路由 |
| `agents/openai.yaml` | `display_name`、`short_description`、`default_prompt` 与 `SKILL.md` 一致；默认提示词显式包含 `$skill-name`；依赖和调用策略不过期 |
| `references/` | 只放 Agent 需要按需阅读的知识/规范；由 `SKILL.md` 直接路由；引用存在；避免与主文重复 |
| `scripts/` | 参数、输出、错误码和依赖明确；至少运行代表性测试；不默认执行破坏性/对外写操作 |
| `assets/` | 只放交付物要复用的模板/资产；可打开、解析或渲染；无个人信息和本地路径；第三方资产有可用权限 |
| `examples/` | 如存在，区分应触发、不应触发、正常 case 和失败 case；不包含私人原始材料 |
| 依赖/工具 | 系统命令、Python/Node 库、CDN、MCP 或外部服务有明确用途、失败兜底和最小权限 |

## 全仓一致性

- 根 README 中每个 Skill 必须存在，每个对外 Skill 也必须出现在目录。
- 安装提示词和命令中的 `--repo`/`--path` 必须与仓库和目录一致。
- 目录树、Skill 名、显示名、默认提示词和文档不得保留旧名。
- 直接引用的所有资源必须存在；孤立资源要删除或接入流程。
- 公开仓库发布前扫描凭据、私钥、个人信息、绝对路径、占位符、缓存和大型非必要文件。

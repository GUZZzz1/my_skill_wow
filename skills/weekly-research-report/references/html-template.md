# HTML 周报模板规范

## 文件命名

默认文件名：

`姓名_工作报告_YYYY_MMDD-MMDD.html`

例如：

`张三_工作报告_2026_0706-0712.html`

## 推荐结构

导师阅读版必须以 `assets/advisor-report-template.html` 为起点，保留当前模板的版式、响应式布局、节点进度、Mermaid 关系图、ECharts 对比/雷达图和 D3 时间轴/里程碑。先复制模板，再替换占位符、示例文案、图表数据、日期和进度状态。

当前模板结构：

1. 工作概览
   - 一句话说明本周状态。
   - 当前阶段判断。
   - 节点进度：小论文线、大论文线。
   - 下周优先级。
2. 本周小论文工作
   - 周初/周中/周末或按任务划分。
   - 小论文状态。
   - 关键结果或表格。
3. 与大论文开题目标的关系
   - 大论文题目。
   - 当前成果对应开题任务的哪一部分。
   - 还缺哪些模块。
4. 大论文后续内容
   - 基础工作。
   - 新增工作一。
   - 新增工作二。
5. 后续工作安排
   - 时间轴。
   - 关键节点。
   - 工作量趋势。
   - 任务、目标、产物、风险表格。

## 可视化选择

不要堆图。按需要选择：

- 节点进度：用 CSS 即可。
- 关系图：用 Mermaid。
- 结果对比和工作量趋势：用 ECharts。
- 时间轴和里程碑：用 D3。

如果只有少量信息，优先文字和表格，不强行画图。

## 写作规则

- 用短句。
- 少用抽象词，如“口径、闭环、沉淀、赋能、抓手、收束”。
- 不把阅读、讨论、计划写成已完成。
- 不把未验证结果写成稳定提升。
- 每个后续任务最好有目标、产物和风险。
- 导师阅读版要便于扫读，不写自我情绪。
- 自我复盘版可以加入“是否拖延、下周验收标准、失败后果”。

## 本地校验

生成或修改 HTML 后至少做：

- HTML parser 检查。
- 内联 JS 语法检查。
- 搜索旧文件名、旧日期、明显残留文案。
- 搜索 `{{...}}` 占位符，交付版不得保留占位符。
